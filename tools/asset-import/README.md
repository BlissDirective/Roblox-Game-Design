# tools/asset-import/

> Notes and helper scripts for the asset upload pipeline.
> Mechanics live in `docs/playbooks/PUBLISHING.md` §4 and
> `docs/02_11_asset_pipeline.md`. This folder is the operational tooling.

**Status:** Stub. Scripts land during Phase G/H when first-party assets need
upload.

---

## Files (planned)

- `upload-manifest.json` — source-of-truth for which local `assets/` files
  map to which Roblox AssetId. Regenerated by the upload workflow.
- `upload-assets.sh` (in `tools/scripts/`) — walks `assets/`, POSTs new files
  to `https://apis.roblox.com/assets/v1/assets`, polls each operation,
  rewrites `src/shared/AssetIds.luau`, opens a PR.

## When to run

- Manual dispatch only (`.github/workflows/upload-assets.yml`,
  `workflow_dispatch`). Roblox AssetIds are immutable — never auto-upload.
- Use `dry_run = true` the first time on any new asset set.

## Out of scope here

- CSG / `EditableMesh` / `EditableImage` assets must be authored in Studio
  and committed manually — Open Cloud cannot publish them remotely. See
  `playbooks/PUBLISHING.md` §3.
